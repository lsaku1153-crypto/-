**项目背景**
- 现在的桌面美化工具很擅长“让桌面变好看”，比如停靠栏、图标隐藏、整体风格统一
- 但它们大多停留在视觉层面，无法真正解决“桌面文件越积越乱”的长期问题
- 传统文件管理工具又太工具化、太冷，不适合生活娱乐赛道，缺少陪伴感和持续使用动力
- 用户真正需要的不是一次性清理，而是一个能陪着自己长期维护桌面状态的智能伙伴

**痛点分析**
- `视觉整洁是假整洁`：桌面文件被隐藏了，但没有被真正整理，时间久了还是会堆积
- `整理动作太打断体验`：用户一旦开始工作、娱乐或学习，通常不想切出去做复杂文件管理
- `传统整理工具缺少情感连接`：它们能管理文件，但不能建立长期使用习惯
- `桌面生态割裂`：美化、文件收纳、提醒、陪伴是分散的，没有形成统一体验

**创意核心**
- 这不是一个“会自动分类的文件工具”
- 这是一个“生活在桌面上的 AI 管家”
- 它先通过沉浸式桌面体验和桌宠互动建立陪伴感，再在用户自然使用电脑的过程中，半自动完成文件识别、收纳建议、场景整理和长期维护
- 用户不是被迫整理桌面，而是在与桌宠互动的过程中，被轻柔地引导完成整理

**目标用户**
- `泛用户`：不局限于办公人群，也不只面向学生或二次元用户
- `适用人群特征`：希望桌面好看、好玩、轻松管理，不想被复杂文件系统打扰
- `使用动机`：先因为桌面氛围和桌宠留下来，再因为整理体验和智能维护持续使用

**核心场景**
- `日常使用`：用户打开电脑，桌宠问候并根据当前状态切换桌面氛围
- `文件堆积`：当桌面截图、安装包、文档、压缩包逐渐变多，系统主动给出轻量整理建议
- `场景切换`：进入学习、办公、娱乐模式时，桌面布局和收纳规则自动变化
- `整理反馈`：文件整理完成后，桌宠用拟人化方式反馈结果，增强成就感
- `长期维护`：系统不是一次性大扫除，而是持续帮用户维持“舒服的桌面状态”

**核心功能**
- `沉浸式桌面层`
支持类似桌面美化工具的视觉体验，让用户的第一感知是“这个桌面很有生命感”
- `桌宠陪伴层`
桌宠负责问候、提醒、夸奖、反馈和轻互动，是情绪陪伴者，也是整理过程中的表达入口
- `AI 文件识别层`
识别文件类型、名称语义、时间、频率和来源，判断哪些文件适合归类、归档或暂存
- `半自动整理机制`
系统提出建议，用户可以一键确认；支持预览、撤销和临时收纳，避免“AI 乱动我文件”的不安全感
- `场景模式`
学习模式、办公模式、娱乐模式、休息模式下，桌面布局、提示语气和整理策略都不同
- `长期桌面维护`

**HTML 做不到的**

| 能力 | 原因 |
|------|------|
| 悬浮在所有窗口之上 | 浏览器被限制在自己的窗口内，无法覆盖其他应用 |
| 点击穿透 | HTML 元素无法把点击事件传递给底层窗口 |
| 系统托盘图标 | 浏览器没有系统级 API |
| 真正的文件管理 | 浏览器沙箱限制，无法操作真实文件系统 |
| 高性能精灵动画 | VPet 是 60fps PNG 序列帧播放 + SkiaSharp 渲染，HTML 的 DOM/CSS 动画做不到那种流畅度 |
| 开机自启、后台常驻 | 浏览器不具备系统服务能力 |
| 多显示器跟随 | 浏览器窗口无法跨屏感知 |

**VPet 是怎么做的**
- C# WPF 桌面应用
- SkiaSharp 渲染引擎逐帧播放 PNG 精灵序列
- 窗口属性设成 `WS_EX_LAYERED` + `WS_EX_TRANSPARENT` 实现置顶 + 点击穿透
- 直接调用 Windows API

**如果你想让这个 Demo 更接近真实产品**

可以升级为 Electron 应用 — 它本质就是把你的 HTML/CSS/JS 包进一个拥有桌面权限的窗口里。你现在写的所有交互代码、SVG 角色、场景切换逻辑大部分可以直接复用，只是加一层窗口控制。但这条路需要额外的开发环境搭建，不适合比赛 Demo 阶段。

**当前 Demo 的定位**
它最适合的角色就是 `概念验证原型` — 证明这个创意成立、交互逻辑跑得通、视觉风格立得住。比赛评委看的是创意和完成度，不是"是不是真的接管了你的 Windows 桌面"。
